The research library contains more than 400 research project comprised of roughly 1500 reports and reports along with all of their associated metadata

There is a fairly complex data model behind the scenes, which is one of Drupal's great strengths

  • Keyword search
  • Advanced filters
  • Search API

Having this data in Drupal allows us to use its powerful Search API and to build multiple methods for searching and linking data together

  • Project nodes
  • Report nodes
  • PDF imports
  • Aggregated data

Because we are using Drupal's content model, we can do all of the traditional things such as individual pages for each project, with all of its related studies, which also get their own unique page

We were also able to do some neat things with the data that weren't possible in InPresto such as using data from individual reports to popuplate aggregated metadata about each project, and to create relationships that were not possible in InPresto

Pretty cool!

  • Powered by Drupal 8
  • Fully customizable
  • 100% automated
  • Seamless user experience

Okay so this all pretty cool, and we now have a full-fledged Drupal 8 site that is seamlessly pulling new data in from Wilder's external research library.
